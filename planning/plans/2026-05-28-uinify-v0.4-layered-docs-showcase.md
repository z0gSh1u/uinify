````mdx
---
title: Layered Public API
description: Choose the core, transcript, composer, adapter, or theme layer for a v0.4 integration.
---

Uinify v0.4 is a layered agent UI kit. Start with the layer that matches the work you are doing, then compose additional layers only when the product needs them.

## Rule

Choose the smallest public layer that solves the integration need: core for state, transcript for rendering, composer for input, adapter for mapping, and theme for visual customization.

## Core Layer

Use the `uinify` package root when you need canonical UI state and events.

Stable core APIs include `createChatRuntime`, `UiRuntimeState`, `UiMessage`, `UiMessagePart`, `UiStreamEvent`, `UiComposerCommand`, and adapter helpers such as `createAdapterRunner`.

Use core when you already have host events and need a normalized transcript model.

```ts
import { createChatRuntime, type UiStreamEvent } from "uinify"

const runtime = createChatRuntime({ conversationId: "layered-demo" })
const events: UiStreamEvent[] = [
  { type: "message.started", messageId: "m1", role: "assistant" },
  { type: "part.text.delta", messageId: "m1", partId: "text-1", delta: "Core owns state and events." },
  { type: "message.completed", messageId: "m1" },
]

for (const event of events) {
  runtime.dispatch(event)
}
```

## Transcript Layer

Use `uinify/react` when you want default rendering or renderer override seams. `ChatRoot`, `MessageList`, `Message`, `MessagePart`, `StepBlock`, `ImagePart`, `ReasoningBlock`, and `ArtifactContainer` live here.

Use transcript overrides such as `renderStep`, `renderImage`, and artifact registry hooks before replacing the whole message list.

```tsx
import { ChatRoot, MessageList } from "uinify/react"

<ChatRoot runtime={runtime} renderers={{ renderStep: ({ part }) => <strong>{part.label}</strong> }}>
  <MessageList />
</ChatRoot>
```

## Composer Layer

Use `uinify/composer/lexical` when the product needs text input, attachments, and command selection.

Commands are input intent. They are not tool results and they are not executed by Uinify.

```tsx
import { LexicalComposer } from "uinify/composer/lexical"

<LexicalComposer
  commands={[
    { id: "agent-research", kind: "agent", label: "Researcher", insertText: "@researcher ", trigger: "@" },
    { id: "mcp-browser-search", kind: "mcp", label: "browser.search", insertText: "/browser.search ", trigger: "/" },
  ]}
  onSubmit={(value) => {
    console.log(value.commands)
  }}
/>
```

## Adapter Layer

Use adapter helpers when a host protocol needs to be converted into canonical `UiStreamEvent[]`.

Adapters stop at UI events plus diagnostics. They do not become vendor SDKs, model clients, or workflow engines.

```ts
import { createAdapterRunner, type UiStreamEvent } from "uinify"

const adaptHostEvent = createAdapterRunner((event: { id: string; text: string }): UiStreamEvent[] => [
  { type: "message.started", messageId: event.id, role: "assistant" },
  { type: "part.text.delta", messageId: event.id, partId: `${event.id}-text`, delta: event.text },
  { type: "message.completed", messageId: event.id },
])
```

## Theme Layer

Use `uinify/styles.css`, CSS tokens, stable `data-slot` regions, and `slotClassNames` when the default structure is right but the visual language needs to change.

Prefer this order: tokens, stable slots, `slotClassNames`, renderer overrides, and only then replacing large transcript components.
````

- [ ] **Step 1: Create the page with step event examples**

Create `docs-site/src/content/docs/integration/agent-steps.mdx` with this content.

````mdx
---
title: Agent Steps
description: Represent agent execution, including tool calls, as canonical transcript steps.
---

Agent work is represented as `step` message parts. A tool call is a step with `category: "tool"`, not a separate canonical tool call model.

## Rule

Map tools, retrieval, handoffs, planners, workflows, skills, and custom execution nodes to `step` parts. Use `category: "tool"` for tool calls.

## Step Part Shape

```ts
type UiStepPart = {
  id: string
  kind: "step"
  category: "tool" | "retrieval" | "handoff" | "planner" | "workflow" | "skill" | "custom"
  status: "pending" | "running" | "complete" | "error"
  label: string
  summary?: string
  inputSummary?: string
  outputSummary?: string
  error?: string
  startedAt?: string
  completedAt?: string
  metadata?: Record<string, unknown>
}
```

## Step Events

Use step lifecycle events to update one part by `messageId` and `partId`.

```ts
import type { UiStreamEvent } from "uinify"

export const stepEvents: UiStreamEvent[] = [
  { type: "message.started", messageId: "assistant-1", role: "assistant" },
  {
    type: "part.step.started",
    messageId: "assistant-1",
    partId: "search-step",
    category: "tool",
    label: "Search docs",
    inputSummary: "query: agent steps",
  },
  {
    type: "part.step.completed",
    messageId: "assistant-1",
    partId: "search-step",
    outputSummary: "Found the Agent Steps guide.",
  },
  { type: "message.completed", messageId: "assistant-1" },
]
```

Use `part.step.updated` for partial progress and `part.step.failed` for recoverable step errors.

```ts
const failedToolStep: UiStreamEvent[] = [
  {
    type: "part.step.started",
    messageId: "assistant-1",
    partId: "browser-step",
    category: "tool",
    label: "browser.search",
    inputSummary: "query: release notes",
  },
  {
    type: "part.step.failed",
    messageId: "assistant-1",
    partId: "browser-step",
    error: "Search backend unavailable",
    outputSummary: "No search results were returned.",
  },
]
```

## Mapping Guidance

Map host tool protocols into step events at the adapter boundary. Do not pass vendor-specific tool payloads into transcript state.

```ts
import type { UiStreamEvent } from "uinify"

type HostToolEvent = {
  messageId: string
  toolCallId: string
  name: string
  input: string
  output?: string
}

export function mapToolCompleted(event: HostToolEvent): UiStreamEvent[] {
  return [
    {
      type: "part.step.started",
      messageId: event.messageId,
      partId: event.toolCallId,
      category: "tool",
      label: event.name,
      inputSummary: event.input,
    },
    {
      type: "part.step.completed",
      messageId: event.messageId,
      partId: event.toolCallId,
      outputSummary: event.output,
    },
  ]
}
```

## Error Handling

A failed step is UI state. Use `status: "error"` through `part.step.failed` when one tool or retrieval node failed but the assistant message can still continue.

Use `message.failed` only when the entire assistant message fails.
````

````mdx
---
title: Multimodal Images
description: Map host-owned image uploads into canonical transcript image parts.
---

Images are transcript message parts. Uploads, validation, object URLs, remote URLs, cancellation, and cleanup remain host-owned.

## Rule

Use the composer to collect image attachments. Use host code to upload or create a local URL. Then dispatch `part.image.emitted` to render an image part in the transcript.

## Composer Submit

```tsx
import { LexicalComposer } from "uinify/composer/lexical"

<LexicalComposer
  onSubmit={(value) => {
    const imageAttachment = value.attachments.find((attachment) => attachment.mimeType.startsWith("image/"))

    if (!imageAttachment) {
      return
    }

    const url = URL.createObjectURL(imageAttachment.file)
    console.log(url, imageAttachment.name)
  }}
/>
```

## Transcript Image Event

After the host owns a displayable URL, dispatch an image part.
The event `partId` becomes the image part `id`; the runtime supplies `kind: "image"`.

```ts
runtime.dispatch({ type: "message.started", messageId: "user-1", role: "user" })
runtime.dispatch({
  type: "part.image.emitted",
  messageId: "user-1",
  partId: "image-1",
  image: {
    url: "https://cdn.example.com/uploads/sketch.png",
    alt: "Uploaded sketch",
    mimeType: "image/png",
    width: 640,
    height: 480,
    sourceAttachmentId: "attachment-1",
  },
})
runtime.dispatch({ type: "message.completed", messageId: "user-1" })
```

## Boundaries

Uinify does not upload files, store blobs, validate file policy, call a model, or revoke object URLs for the host.

If host code uses `URL.createObjectURL`, do not revoke immediately after dispatch. Revoke when the rendered image is removed or the component unmounts.

```ts
const url = URL.createObjectURL(file)
const activeObjectUrls = new Set<string>()
activeObjectUrls.add(url)

runtime.dispatch({
  type: "part.image.emitted",
  messageId: "user-1",
  partId: "image-1",
  image: { url, alt: file.name, mimeType: file.type },
})

// Later, when the rendered image is removed or the component unmounts:
for (const activeUrl of activeObjectUrls) {
  URL.revokeObjectURL(activeUrl)
}
activeObjectUrls.clear()
```

Use attachment lifecycle state for selection, validation, upload progress, retry, and cancellation. Use image parts only for transcript rendering.
````
