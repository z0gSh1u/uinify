# Uinify v0.4 Transcript Agent UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add default React transcript primitives for agent steps and image parts while removing tool-call-specific rendering from the recommended path.

**Architecture:** Build on the canonical `UiStepPart` and image event model from the first v0.4 sub-plan. Add a focused `StepBlock`, wire it through `MessagePart`, add renderer override support, and stabilize slots/styles for step and image rendering.

**Tech Stack:** React, TypeScript, Testing Library, Vitest, react-virtuoso test mocks, existing renderer context.

---

## Assumptions

- `planning/plans/2026-05-28-uinify-v0.4-canonical-agent-primitives.md` has already been implemented.
- `src/model/types.ts` exports `UiStepPart`, `UiStepCategory`, `UiStepStatus`, and an expanded `UiImagePart` with `mimeType`, `width`, `height`, and `sourceAttachmentId`.
- `UiMessagePart` no longer includes `UiToolCallPart`, and `src/runtime/events.ts` no longer includes `part.tool.updated`.
- This plan must not reintroduce `ToolCallBlock`, `UiToolCallPart`, `renderToolCall`, `tool-call`, or `part.tool.updated` in active recommended rendering paths.

```tsx
import type { UiStepPart } from "../model/types"

export type StepBlockProps = {
  part: UiStepPart
}

export function StepBlock({ part }: StepBlockProps) {
  return (
    <section data-category={part.category} data-slot="step" data-state={part.status}>
      <header data-slot="step-header">
        <div data-slot="step-title">{part.label}</div>
        <div data-slot="step-badges">
          <span data-slot="step-category">{part.category}</span>
          <span data-slot="step-status">{part.status}</span>
        </div>
      </header>
      {part.summary ? <p data-slot="step-summary">{part.summary}</p> : null}
      {part.inputSummary ? (
        <div data-slot="step-input">
          <span data-slot="step-detail-label">Input</span>
          <p>{part.inputSummary}</p>
        </div>
      ) : null}
      {part.outputSummary ? (
        <div data-slot="step-output">
          <span data-slot="step-detail-label">Output</span>
          <p>{part.outputSummary}</p>
        </div>
      ) : null}
      {part.error ? <p data-slot="step-error">{part.error}</p> : null}
    </section>
  )
}
```

Replace `src/react/image-part.tsx` with this implementation.

```tsx
import { useState } from "react"
import type { UiImagePart } from "../model/types"

export type ImagePartProps = {
  part: UiImagePart
}

export function ImagePart({ part }: ImagePartProps) {
  const [failed, setFailed] = useState(false)

  return (
    <figure
      data-mime-type={part.mimeType}
      data-slot="image"
      data-source-attachment-id={part.sourceAttachmentId}
      data-state={failed ? "error" : "loaded"}
    >
      {failed ? (
        <figcaption data-slot="image-fallback">Image failed to load</figcaption>
      ) : (
        <img
          alt={part.alt ?? ""}
          height={part.height}
          onError={() => setFailed(true)}
          src={part.url}
          width={part.width}
        />
      )}
      {part.alt ? <figcaption data-slot="image-caption">{part.alt}</figcaption> : null}
    </figure>
  )
}
```
