# Uinify v0.4 Composable Agent UI Kit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Decompose the approved v0.4 Composable Agent UI Kit design into focused, independently executable implementation plans.

**Architecture:** The approved design spans five subsystems: canonical model/runtime primitives, transcript rendering, composer commands, multimodal composer-to-transcript flow, and docs/showcase. This umbrella plan intentionally avoids one oversized implementation pass. Each sub-project gets its own implementation plan so changes can be reviewed, tested, and committed independently.

**Tech Stack:** TypeScript, React, Lexical, Vitest, Testing Library, tsup, Astro Starlight, Vite example playground.

---

## Scope Decision

The design in `planning/specs/2026-05-28-uinify-v0.4-composable-agent-ui-kit-design.md` covers multiple independent subsystems. A single implementation plan would be too broad and would make review risky.

Use this umbrella plan to create and execute five focused implementation plans in order:

1. `planning/plans/2026-05-28-uinify-v0.4-canonical-agent-primitives.md`
2. `planning/plans/2026-05-28-uinify-v0.4-transcript-agent-ui.md`
3. `planning/plans/2026-05-28-uinify-v0.4-composer-command-primitives.md`
4. `planning/plans/2026-05-28-uinify-v0.4-multimodal-composer-transcript.md`
5. `planning/plans/2026-05-28-uinify-v0.4-layered-docs-showcase.md`

Each sub-plan should start from tests, produce working software on its own, and end with focused verification.
