---
title: Stable Slots
description: Durable data-slot regions and slotClassNames for targeted styling.
---

`uinify` exposes stable `data-slot` regions and matching `slotClassNames` keys so hosts can style the UI without depending on incidental DOM nesting.

## Common Slots

- `message`
- `message-parts`
- `message-actions`
- `part-actions`
- `artifact-container`
- `artifact-tabs`
- `artifact-views`
- `attachment-tray`
- `attachment-actions`
- `attachment-part`

## Practical Usage

Use slots when token-level theming is not specific enough:

```css
[data-slot="message-actions"] {
  justify-content: flex-end;
}

[data-slot="artifact-tabs"] {
  border-bottom: 1px solid color-mix(in srgb, currentColor 18%, transparent);
}
```

If you are passing `slotClassNames`, keep the names aligned with the documented slot keys instead of inventing a second selector contract.

## Recommendation

Treat documented slots as the stable styling API. Avoid selectors that depend on internal wrapper depth when a slot already exists for the same region.
