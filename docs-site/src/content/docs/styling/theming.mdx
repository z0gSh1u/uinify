title: Theming
description: Override CSS custom properties to match your app's visual style.
---

Start by importing `uinify/styles.css` once, then override the exposed `--uinify-*` CSS custom properties in your app stylesheet.

## Recommended Pattern

```css
:root {
  --uinify-color-bg: #0f172a;
  --uinify-color-fg: #e5e7eb;
  --uinify-color-accent: #7c3aed;
}
```

You can scope overrides to part of the page instead of `:root` when the chat UI should inherit a local theme:

```css
.support-chat {
  --uinify-color-bg: #111827;
  --uinify-color-fg: #f9fafb;
}
```

## Practical Order

1. Import the base stylesheet.
2. Override tokens for color, spacing, radius, and typography.
3. Use documented slots when a specific region needs targeted rules.
4. Replace renderers only when the product shape itself must change.

For long-lived styling hooks, pair this page with [Stable Slots](/styling/slots/).
