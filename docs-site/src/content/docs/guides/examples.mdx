The repository no longer keeps speculative app templates. The example surface is intentionally narrow: one runnable Vite examples app with one real OpenAI-compatible chat route.

## Current Files

- `examples/src/App.tsx`: local examples app shell and route registry
- `examples/src/chat/ChatExample.tsx`: `/chat` route that sends text prompts to the local Vite API proxy and renders streamed replies with `ChatRoot` plus `MessageList`
- `examples/server/openai-compatible-chat.ts`: dev-server adapter that reads `.env`, calls an OpenAI-compatible Chat Completions stream, and emits canonical `UiStreamEvent` SSE frames

Run the examples app locally:

```bash
pnpm dev:example
```

The real chat route expects these local environment variables:

```bash
UINIFY_LLM_BASE_URL=https://api.openai.com/v1
UINIFY_LLM_API_KEY=replace-me
UINIFY_LLM_MODEL=gpt-4o-mini
```

## How To Read The Example

Use `/chat` to verify a real text-only chat flow. The browser posts messages to `/api/chat`; the Vite dev server keeps the API key server-side, maps the provider stream into `UiStreamEvent`, and streams those events back to the page.

Do not treat the dev-server adapter as a supported OpenAI compatibility layer. Real products should map their own backend payloads into `UiStreamEvent` at the host boundary.
