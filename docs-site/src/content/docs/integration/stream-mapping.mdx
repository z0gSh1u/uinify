title: Stream Mapping
description: Map your backend protocol into UiStreamEvent before it reaches the runtime.
---

Map your host or backend protocol into `UiStreamEvent` before it reaches the runtime. This keeps `uinify` focused on chat state and rendering instead of vendor-specific transport details.

The examples app keeps the concrete mapping logic in `examples/server/openai-compatible-chat.ts`, where the local dev server maps an OpenAI-compatible stream into canonical `UiStreamEvent` frames.

If your transport is Server-Sent Events, pair this page with [SSE Integration](/integration/sse/). For the runnable examples app, see [Examples](/guides/examples/).

For first-time adoption, the recommended default setup is to import `uinify/styles.css` once near your app entrypoint.

## Pattern

```ts
import type { UiStreamEvent } from "uinify"

type HostEvent =
  | { kind: "begin"; id: string }
  | { kind: "text"; id: string; partId: string; text: string }
  | { kind: "end"; id: string }

export function mapHostEvent(event: HostEvent): UiStreamEvent[] {
  switch (event.kind) {
    case "begin":
      return [
        {
          type: "message.started",
          messageId: event.id,
          role: "assistant",
        },
      ]

    case "text":
      return [
        {
          type: "part.text.delta",
          messageId: event.id,
          partId: event.partId,
          delta: event.text,
        },
      ]

    case "end":
      return [
        {
          type: "message.completed",
          messageId: event.id,
        },
      ]
  }
}
```

Dispatch the mapped events into the runtime:

```ts
for (const streamEvent of mapHostEvent(event)) {
  runtime.dispatch(streamEvent)
}
```

## Agent Steps And Images

Tool calls and other agent execution nodes map to step events. A tool call is a `step` with `category: "tool"`.

```ts
import type { UiStreamEvent } from "uinify"

type HostToolCompleted = {
  kind: "tool.completed"
  messageId: string
  toolCallId: string
  name: string
  input: string
  output: string
}

export function mapToolCompleted(event: HostToolCompleted): UiStreamEvent[] {
  return [
    {
      type: "part.step.started",
      messageId: event.messageId,
      partId: event.toolCallId,
      category: "tool",
      label: event.name,
      inputSummary: event.input,
    },
    {
      type: "part.step.completed",
      messageId: event.messageId,
      partId: event.toolCallId,
      outputSummary: event.output,
    },
  ]
}
```

Images map to transcript image parts after the host has a displayable URL.
The `partId` supplies the image part `id`; the runtime supplies `kind: "image"`.

```ts
const imageEvent: UiStreamEvent = {
  type: "part.image.emitted",
  messageId: "user-1",
  partId: "image-1",
  image: {
    url: "https://cdn.example.com/sketch.png",
    alt: "Uploaded sketch",
    mimeType: "image/png",
    sourceAttachmentId: "attachment-1",
  },
}
```

## Stable Adapter Contract

When your mapper becomes a shared integration surface, wrap the pure mapper with `createAdapterRunner`. That gives you a stable result shape with canonical `.events` plus `.diagnostics` for suspicious ordering and invalid artifact views.

```ts
import { createAdapterRunner, type UiStreamEvent } from "uinify"

type HostEvent =
  | { kind: "begin"; id: string }
  | { kind: "text"; id: string; partId: string; text: string }
  | { kind: "end"; id: string }

function mapHostEvent(event: HostEvent): UiStreamEvent[] {
  switch (event.kind) {
    case "begin":
      return [{ type: "message.started", messageId: event.id, role: "assistant" }]
    case "text":
      return [{ type: "part.text.delta", messageId: event.id, partId: event.partId, delta: event.text }]
    case "end":
      return [{ type: "message.completed", messageId: event.id }]
  }
}

const adaptHostEvent = createAdapterRunner(mapHostEvent)

for (const result of incomingEvents.map(adaptHostEvent)) {
  if (result.diagnostics.length > 0) {
    console.warn(result.diagnostics)
  }

  for (const streamEvent of result.events) {
    runtime.dispatch(streamEvent)
  }
}
```

## Rule

`message.started` must arrive before any part events for that message. The runtime does not buffer orphan part events.

Mapping ends at `UiStreamEvent`. Do not pass vendor-specific payloads or transport-specific event objects into `uinify` state. Use `step` events for tools and agent execution, and use `part.image.emitted` only after host code has a displayable image URL.

Use `createAdapterResult` and `validateAdapterEvents` directly only if `createAdapterRunner` is not enough for your wrapper.
